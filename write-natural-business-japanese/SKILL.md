# 中文 → 自然な商務日本語

三层分工，各管各的：

| 层 | 原则 | 自由度 |
|---|---|---|
| **文章构型** | 与中文版镜像：章节顺序、表格框架、条目数、编号、数字一一对应 | 零（保证双语版本能逐节对照、同步维护） |
| **句子** | 先确定这句中文要传达的意思，再丢开原文句法，用日语把这个意思说出来 | 高：可拆句、换语序、换主语、加日语缓冲表达；不得增删信息点 |
| **词汇** | 照搬进日语的每个汉语词都过一遍「三张陷阱表」 | 判定标准唯一：日本人原生写同类文档会用这个词吗 |

为什么分层：结构动了，双语版本无法对照、后续同步必漏；句子不放开，必然残留中文句法；词汇不过表，句法再自然也会踩「質量」「登記」这类一眼露馅的雷。

## 工作流程

1. **定骨架**：照搬中文版的标题层级、表格框架、条目顺序。标题译成自然日语，但数量和位置不动。
2. **逐段翻案**：一段一段处理。先在心里说出这段的意思，然后用日语直接写这个意思。写完回读：若能和中文逐词对上，说明是词语替换而不是重写，重来。
3. **三表扫描**：成稿后对全文做一遍词汇审查（下面表 A/B/C）。
4. **收尾**：文体检查 + 自检清单。

## 表 A：同形异义汉语词

### A-1 意思完全不同（高危，一眼露馅）

| 中文 | 同形日语词的真实含义 | 应该写 |
|---|---|---|
| 质量 | 質量＝物理质量（mass） | 品質 |
| 工作 | 工作＝手工、（暗中）活动 | 業務、作業 |
| 结束 | 結束＝团结 | 終了、完了 |
| 项目 | 項目＝条目、字段 | プロジェクト、案件 |
| 经理 | 経理＝会计 | マネージャー、〜長 |
| 登录 | 登録＝注册 | ログイン |
| 需求 | 需要＝（经济学）需求 | 要件、要望、ニーズ |
| 告诉 | 告訴＝刑事告诉 | 伝える、連絡する |
| 单位（机构） | 単位＝计量单位、学分 | 組織、部署、所属先 |
| 登记 | 登記＝不动产・法人登记 | 登録 |
| 检讨 | 検討＝研究考虑 | 反省、見直し |

### A-2 语域错位或根本不是日语（中危，读着别扭）

| 中文习惯 | 问题 | 自然日语 |
|---|---|---|
| 口径（对外口径） | 日语只有物理口径义 | 説明方針、言い回し、〜の扱い |
| 保底 | 不是日语 | 予備、フォールバック、最低保証 |
| 总机 | 総機不是日语 | 受付、窓口、代表電話 |
| 人工〜 | 人工＝人造 | 有人〜、手動〜、担当者による〜 |
| 主演（比喻） | 只指演员主演 | メイン、目玉 |
| 优化 | 優化不是日语 | 最適化、改善 |
| 完善（动词） | 完善不作动词用 | 整備する、充実させる |
| 落实 | 不是日语 | 徹底する、実行に移す |
| 安排 | 不是日语 | 手配、段取り、調整 |
| 汇报 | 不是日语 | 報告 |
| 方案 | 日语非常用词 | 案、プラン、方式 |
| 运行（系统） | 運行＝交通工具按班次运行 | 稼働、動作 |
| 升级（系统） | 昇級＝人的晋升 | アップグレード、バージョンアップ |
| 维护（系统） | 維護不是日语 | 保守、メンテナンス |
| 界面 | 界面＝（化学）界面 | 画面、UI、インターフェース |

### A-3 不是陷阱（不要过度矫正）

対応、確認、案件、実施、把握、措置、現場、架電、受電、入電、折返し——这些是地道日语（后四个是呼叫中心行话），照用即可。

注意「地道但别用错槽位」：検収是地道日语，但专指对**交付物**的验收（納品物の受入検査）；「实机验收通过」要写「実機検証に合格」，不是「実機で検収」。

## 表 C：中式句法与标点

- 名词长定语链（「AI完了偽装防止機構」式堆叠）→ 拆句，或用「〜するための〜」。
- 「〜を実施／進行／展開する」官腔 → 用具体动词：行う、進める、始める。
- 四字格对仗（安全可靠、高效稳定）→ 日语不追求对仗，选一个准确的词。
- 一逗到底的长句 → 一句一义，多用句号。
- 中文标点残留（，；：""《》）→ 日语标点（、。「」『』）；冒号只用于引导列表，不进句子。
- 「本次／该」→ 今回の、この。
- 量词直搬（2 周連続、三本场景）→ 按日语习惯：2 回通し、シナリオ 3 本。

## 文体与语域

| 场景 | 文体 |
|---|---|
| 客户・营业方向 | です・ます体；お客様、ご確認、ご提案等美化语；结论先行 |
| 内部规格・清单 | だ・である体或体言止め；术语可保留英文 |
| 邮件 | です・ます体＋定型寒暄（お世話になっております等） |

- 对外文档出现内部管理编号时必须带一句人话说明（例：管理番号 AI-30＝通話フローへの本人確認組み込み）。
- 敬语拿不准时降级用丁寧語，不硬上尊敬语・谦让语——用错比不用更刺眼。
- 片假名外来语取舍以日本同行业文档的高频写法为准，不自创。
- 表格单元格优先体言止め（「予備録画の準備」而非「予備録画を準備する」），比整句更像日本人写的表。
