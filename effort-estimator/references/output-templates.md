# 输出格式模板

## 格式选择指南

| 场景 | 格式 | 特徴 |
|------|------|------|
| 顧客への正式見積 | 日式見積書 | 工程×工数×単価、諸経費、月額 |
| 内部技術検討 | WBS詳細表 | タスク分解、依存関係、PERT期待値 |
| 初回打診・概算 | 一頁摘要 | モジュール別大枠、前提条件明記 |
| 予算申請 | Phase分割表 | フェーズ別、リスクバッファ明示 |
| チーム体制提示 | 人員配置表 | スキル係数、アサイン期間 |

---

## 一、日式見積書テンプレート

```markdown
# 見積書

## 1. はじめに
本見積書は、{案件名}に関するシステム開発費用を提示するものです。

## 2. 見積もり条件

### 前提条件
- 本見積もりは{要件書名/打ち合わせ内容}に基づきます
- 開発方式：{フルスタック1名 / チームX名}
- ご提供いただくもの：{kintone環境 / API キー / 用語辞書 等}

### 対象範囲外（将来フェーズ）
- {商材AI提案機能 / ダッシュボード / 管理UI 等}

## 3. 費用内訳

### 3.1 初期開発費用

| 工程 | 作業内容 | 工数(人日) | 単価(円/人日) | 金額(円) |
|------|---------|-----------|-------------|---------|
| 要件定義 | ヒアリング、要件整理 | X | XX | XXX |
| 基本設計 | 画面設計、DB設計、API設計 | X | XX | XXX |
| 開発 | フロントエンド + バックエンド | X | XX | XXX |
| テスト | 単体 + 結合 + 受入テスト支援 | X | XX | XXX |
| **合計** | | **XX人日** | | **XXX円** |

### 3.2 月額費用（ランニングコスト）

| 項目 | 月額(円) | 備考 |
|------|---------|------|
| AI API利用料 | XX | {Whisper, GPT-4, Embedding} |
| クラウドインフラ | XX | {AWS/GCP/Azure} |
| 保守・運用 | XX | 障害対応、軽微改修、問合せ対応 |
| **月額合計** | **XX円** | |

## 4. スケジュール

- 開始：{YYYY年MM月}
- リリース目標：{YYYY年MM月}
- {工程別マイルストン}

## 5. 支払条件

- 初期開発費：{契約時50% + 検収時50%}
- 月額費用：{月末締め翌月末払い}

## 6. 特記事項

- 本見積もりの有効期限：{提出から60日間}
- 仕様変更が発生した場合は別途お見積もり
- 瑕疵担保期間：{検収後6ヶ月}
```

## 五、AI/LLM案件専用 補足テンプレート

LLM案件は不確実性が高いため、通常のWBSに加えて以下を追記：

```markdown
## LLM関連の不確実性評価

| 項目 | 確度 | 変動要因 |
|------|------|---------|
| プロンプト設計・チューニング | 低〜中 | 試行錯誤の反復が必要 |
| AI応答品質目標達成 | 中 | ユーザーFB次第 |
| 音声認識精度 | 中 | 用語辞書の質に依存 |
| RAG検索精度 | 低〜中 | Embeddingモデル/チャンク戦略次第 |
| AI APIコスト | 低 | 利用量の予測困難、実測ベースで調整 |

## 推奨アプローチ
- Phase 1はPoCとして小規模スタート
- 2〜3週間の検証期間後に本開発の見積もりを再調整
- 三点見積法で各AI関連タスクを独立評価
```
